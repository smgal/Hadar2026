# HDPlayer 구조 분석

`HDPlayer`는 각 파티원을 나타내는 개체 모델로, 직업, 속성, 현재 체력 및 마력 상태를 관리합니다.

## 1. 주요 구조 (Flutter)

`lib/models/hd_player.dart`에 위치합니다.

### 주요 속성
| 속성명 | 타입 | 설명 |
| :--- | :--- | :--- |
| `name` | `String` | 캐릭터 이름 (한국어 조사 '은/는', '이/가' 처리를 위한 종성 판별 포함). |
| `gender` | `int` | 0: 남성, 1: 여성. |
| `characterClass` | `int` | 0: 에스퍼, 1: 싸이보그, 2: 초능력자. |
| `strength` ~ `luck` | `int` | 힘(STR), 정신력(INT), 지구력(END) 등 파라미터. |
| `ac` | `int` | 방어력 (Armor Class, 낮을수록 좋음). |
| `hp`, `sp`, `esp` | `int` | 체력, 마력, 초능력의 현재치 및 최대치. |
| `experience` | `int` | 현재 획득한 경험치 총량. |
| `level`, `accuracy` | `List<int>` | [기본, 마법, ESP] 레벨 및 명중률. |
| `poison` ~ `dead` | `int` | 상태 이상 타이머 (0: 정상, >0: 해당 효과 발생 중). |

### 주요 메서드
- `isAvailable()`: 플레이어가 행동 가능한 상태인지 확인 (죽음/의식불명 제외).
- `damaged(int damage)`: 대미지를 입고 HP 감소 및 의식불명 상태를 업데이트합니다.
- `checkLevelUp()`: 누적 경험치를 경험치 테이블과 비교하여 레벨업 및 능력치 성장을 처리합니다.
- `assignFromEnemyData(int enemyId)`: 적 템플릿 데이터로부터 플레이어 스탯을 할당하는 특수 생성자입니다 (스크립트 명령 처리에 중요).
- `getAttribute(String attr)`: 스크립트 엔진에서 통합적으로 속성을 조회하기 위한 게터 인터페이스입니다.

## 2. Unity 원본(`ObjPlayer.cs`)과의 비교

Flutter `HDPlayer`는 기존 C# 버전보다 현저히 평탄화되고 접근이 쉬운 구조로 설계되었습니다.

### 구조적 차이점
- **상속 vs 평탄화**:
  - **Unity**: 이름 관리와 직렬화를 위해 `ObjNameBase`를 상속받았습니다.
  - **Flutter**: 독립적인 클래스로 구성되며, 이름 설정 시 유니코드 연산을 통해 직접 종성을 계산하여 조사를 자동 생성합니다.
- **능력치 관리**:
  - **Unity**: 열거형(Enum)으로 인덱싱된 `int[] status`와 `int[] skill` 배열을 사용했습니다.
  - **Flutter**: `strength`, `agility` 등 명시적인 명칭의 필드를 사용하여 가독성을 높였습니다.
- **장비 복잡도**:
  - **Unity**: 각 장착 슬롯에 상태 보정치와 리소스 ID가 포함된 복잡한 `Equiped` 객체를 가졌습니다.
  - **Flutter**: 현재는 무기/방어구 아이템의 ID(`weapon`, `armor`)와 기본 공격력/방어력(`powOfWeapon` 등) 값을 직접 보유합니다.

### 기능적 변천
- **장비 적용(`Apply`)**: 
  - **Unity**: 장비가 변경될 때마다 `Apply()`를 호출해 순수 능력치와 장비 보정치를 합산하고 파생 스탯(`atta_pow`, `ac`)을 재계산했습니다.
  - **Flutter**: 현재는 간단한 속성 합산 방식으로 구현되어 있습니다.
- **레벨업 및 성장 로직**: 두 버전 모두 거의 동일한 경험치 테이블과 성장 공식을 사용합니다. Flutter의 `1 + (strength ~/ 10)` 성장 공식은 원작의 로직을 그대로 반영합니다.
- **한국어 조사 처리(Josa)**:
  - **Unity**: `ObjNameBase`와 별도의 외부 문자열 리소스 매니저에서 처리했습니다.
  - **Flutter**: `name` 세터 내에서 직접 한글 조사를 관리하도록 내장되었습니다.

> [!TIP]
> Flutter 구조는 현대적인 JSON 형식으로 저장하기에 더 적합하며, Unity 구조는 바이너리 스트림 기반의 정적인 저장 방식에 특화되어 있었습니다.
